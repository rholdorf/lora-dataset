Create new FileNode.swift with a recursive tree structure for folders:

```swift
struct FileNode: Identifiable, Hashable {
    let id = UUID()
    let url: URL
    var children: [FileNode]?  // nil = not loaded, [] = empty, [items] = loaded

    var name: String { url.lastPathComponent }
    var isDirectory: Bool { children != nil }

    var icon: String {
        if children == nil { return "doc" }
        return (children?.isEmpty == true) ? "folder" : "folder.fill"
    }
}
```

Key details:
- children is optional: nil means folder not yet scanned (lazy loading), empty array means scanned but empty
- Use SF Symbols for icons (folder, folder.fill, doc)
- Hashable based on id only (for List selection)
- Add static func == based on id for proper selection behavior

Extend DatasetViewModel with folder tree functionality:

1. Add published properties:
   - `@Published var folderTree: [FileNode] = []` - root level folders
   - `@Published var selectedFolderID: UUID? = nil` - currently selected folder in tree

2. Add helper to get selected folder URL:
   - `var selectedFolderURL: URL?` - computed property returning URL for selectedFolderID

3. Add method `buildFolderTree(from url: URL, depth: Int = 1) -> [FileNode]`:
   - Use FileManager.contentsOfDirectory to get items
   - Filter to directories only (using isDirectoryKey)
   - Skip hidden files (.skipsHiddenFiles)
   - Sort alphabetically by name
   - For depth > 0, recursively load children; for depth = 0, set children to empty array (allows expand)
   - Return sorted array of FileNode

4. Modify `scanDirectory(_:)` to also build folder tree:
   - After scanning pairs, call `folderTree = buildFolderTree(from: folder, depth: 1)`
   - This gives one level of subfolder visibility on initial load

5. Add method `loadChildrenForNode(nodeID: UUID)`:
   - Find node in tree by ID (recursive search)
   - Load children using buildFolderTree with depth: 1
   - Update the node in tree with loaded children
   - This enables lazy loading when user expands a folder

6. Add method `selectFolder(_ url: URL)`:
   - Set directoryURL to new folder
   - Call scanDirectory to reload pairs for that folder
   - Keep same securedDirectoryURL (parent bookmark covers subdirs)

Key: DO NOT create new security-scoped bookmarks for subdirectories - parent bookmark already grants access to all children.
