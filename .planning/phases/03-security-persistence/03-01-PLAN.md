<objective>
Persist folder tree expansion state between app sessions.

Purpose: Users shouldn't need to re-expand folders every time they launch the app. The sidebar should remember which folders were expanded.
Output: Folder expansion state saved to UserDefaults and restored on launch.
</objective>

**Prior decisions affecting this phase:**
- Phase 01-02: OutlineGroup used for folder tree (needs replacement with DisclosureGroup for expansion control)
- Phase 01-02: FileNode uses UUID() - cannot match across sessions, must use path for persistence

**Key insight:** SwiftUI's OutlineGroup doesn't expose expansion state bindings. To persist expansion, we need:
1. Track expanded paths (Set<String>) in ViewModel
2. Replace OutlineGroup with recursive DisclosureGroup that binds to tracked state
3. Persist to UserDefaults on change, restore on launch

<task type="auto">
  <name>Task 1: Add expansion state tracking to ViewModel</name>
  <files>lora-dataset/lora-dataset/DatasetViewModel.swift</files>
  <action>
Add expansion state management to DatasetViewModel:

1. Add `@Published var expandedPaths: Set<String> = []` property
2. Add `toggleExpanded(path: String)` method that adds/removes from set
3. Add `isExpanded(path: String) -> Bool` helper
4. In `init()`, restore from UserDefaults key "expandedFolderPaths" (decode as [String] array, convert to Set)
5. Add `private func persistExpandedPaths()` that saves to UserDefaults
6. Call `persistExpandedPaths()` in `toggleExpanded()` after modifying set

**Important:** Use path strings (url.path) not URLs for UserDefaults compatibility.

<task type="auto">
  <name>Task 2: Replace OutlineGroup with recursive DisclosureGroup</name>
  <files>lora-dataset/lora-dataset/ContentView.swift</files>
  <action>
Replace the OutlineGroup in ContentView with a recursive DisclosureGroup that binds to expandedPaths:

1. Create a new `FolderTreeView` view that takes a `[FileNode]` array and the ViewModel
2. In FolderTreeView, use ForEach over nodes
3. For each node with children, use DisclosureGroup with:
   - isExpanded binding: `Binding(get: { vm.isExpanded(path: node.url.path) }, set: { if $0 { vm.toggleExpanded... } })`
   - Label: FolderRowView (existing)
   - Content: recursive FolderTreeView for node.children
4. For leaf nodes (children == nil or empty), show just the FolderRowView
5. Replace the OutlineGroup in "Pastas" section with FolderTreeView(nodes: vm.folderTree, vm: vm)

**Note:** DisclosureGroup requires explicit toggle handling. The binding set closure should call `vm.toggleExpanded(path:)`.

**Avoid:** Don't create circular view updates - use Task wrapper if needed for state changes.
