**New properties** (add after the existing `qlKeyMonitor` property):

```swift
// MARK: - Filesystem Watchdog
private var contentWatcher: DirectoryWatcher?
private var treeWatcher: DirectoryWatcher?
private var isRescanning = false  // guard against bounce-back loops (Pitfall 2 from RESEARCH.md)
```

**New method: `startWatching(_:rootURL:)`**

Called after security-scoped access is confirmed. Creates two DirectoryWatcher instances on a shared serial utility queue. The content watcher watches `directoryURL` (fires `performContentRescan`). The tree watcher watches `rootDirectoryURL` (fires `performTreeRescan`). Both use default 0.5s debounce.

```swift
private func startWatching() {
    stopWatching()
    guard let contentURL = directoryURL, let rootURL = rootDirectoryURL else { return }
    let q = DispatchQueue(label: "com.loradataset.watchdog", qos: .utility)
    contentWatcher = DirectoryWatcher(url: contentURL, queue: q) { [weak self] in
        Task { @MainActor [weak self] in
            self?.performContentRescan()
        }
    }
    // Only create separate tree watcher if root differs from content directory
    if rootURL.path != contentURL.path {
        treeWatcher = DirectoryWatcher(url: rootURL, queue: q) { [weak self] in
            Task { @MainActor [weak self] in
                self?.performTreeRescan()
            }
        }
    }
    contentWatcher?.start()
    treeWatcher?.start()
    print("[watchdog] watching content: \(contentURL.lastPathComponent), root: \(rootURL.lastPathComponent)")
}
```

**New method: `stopWatching()`**

```swift
private func stopWatching() {
    contentWatcher?.stop()
    contentWatcher = nil
    treeWatcher?.stop()
    treeWatcher = nil
}
```

**New method: `performContentRescan()`**

This is the core diff logic. Guard against re-entrancy with `isRescanning`. Capture old pairs, call `scanCurrentDirectory()`, diff old vs new, handle cache eviction, selection repair, and prefetch re-trigger.

Key logic:
1. `guard !isRescanning else { return }` -- bounce-back guard
2. `isRescanning = true; defer { isRescanning = false }`
3. Capture `let oldPairs = pairs` and `let oldURLs = Set(oldPairs.map(\.imageURL))`
4. Check if `directoryURL` still exists via `FileManager.default.fileExists(atPath:)`. If not, call `navigateToSurvivingAncestor()` and return (per locked decision: navigate to parent on folder deletion).
5. Capture current `selectedID` and the old index of the selected pair. Find the old selected pair's `imageURL` for matching after rescan (since UUIDs are regenerated).
6. Call the existing `scanCurrentDirectory()` -- this updates `self.pairs` and sets `selectedID` to `pairs.first?.id`
7. Compute `let newURLs = Set(pairs.map(\.imageURL))`
8. `let removed = oldURLs.subtracting(newURLs)`
9. `let added = newURLs.subtracting(oldURLs)`
10. For each `url in removed`: call `Task { await imageCache.remove(for: url) }`, cancel and remove from `prefetchTasks[url]`
11. Invalidate cache for files that exist in both old and new but may have changed content -- check by comparing file modification dates would be complex; instead, for URLs present in both sets, leave them cached (LRU will handle staleness). This is a pragmatic choice. For the "file replaced externally" case (per locked decision), evict URLs where the file exists in both old and new sets AND the image was modified. Since detecting modification requires stat() calls, use a simpler approach: only evict removed URLs. The "same filename different content" case is better handled in Phase 12 with per-file VNODE watching.
12. **Selection repair** (per locked decisions):
    - If `oldSelectedURL` (the imageURL of the selected pair before rescan) still exists in new pairs: restore `selectedID = pairs.first(where: { $0.imageURL == oldSelectedURL })?.id`. Also restore `detailID`.
    - If `oldSelectedURL` was deleted (not in new pairs by URL): find the old index of the deleted pair in `oldPairs`, then select the pair at the same index in `newPairs` (clamped to bounds), or `newPairs.last?.id`, or `nil` if empty. Set both `selectedID` and `detailID`. Set `_editingIsDirty = false` (silently discard edits per locked decision).
    - Note: `scanCurrentDirectory()` generates new UUIDs for all pairs. So after rescan, we need to find the pair with the same `imageURL` as the previously selected pair and set `selectedID` to its new UUID.
13. Re-trigger prefetch: `if let id = selectedID { triggerPrefetch(aroundID: id) }`
14. Also rebuild folder tree: `if let root = rootDirectoryURL { folderTree = buildFolderTree(from: root) }` -- this ensures new subfolders under content dir appear in the tree too.
15. `print("[watchdog] content rescan: +\(added.count) -\(removed.count)")`

**New method: `performTreeRescan()`**

Simpler -- just rebuilds the folder tree. Also check if `directoryURL` still exists (folder may have been deleted from within the tree).

```swift
private func performTreeRescan() {
    guard let root = rootDirectoryURL else { return }
    folderTree = buildFolderTree(from: root)

    // If the current content directory was deleted, navigate to surviving ancestor
    if let contentURL = directoryURL,
       !FileManager.default.fileExists(atPath: contentURL.path) {
        navigateToSurvivingAncestor(from: contentURL)
    }
    print("[watchdog] tree rescan complete")
}
```

**New method: `navigateToSurvivingAncestor(from:)`**

Per locked decision: walk up to nearest surviving ancestor within root scope.

```swift
private func navigateToSurvivingAncestor(from url: URL) {
    let fm = FileManager.default
    var candidate = url.deletingLastPathComponent()
    while !fm.fileExists(atPath: candidate.path) && candidate.path != "/" {
        candidate = candidate.deletingLastPathComponent()
    }
    // Stay within root scope
    if let root = rootDirectoryURL,
       candidate.path.hasPrefix(root.path),
       fm.fileExists(atPath: candidate.path) {
        navigateToFolder(candidate)
    } else if let root = rootDirectoryURL {
        navigateToFolder(root)
    }
}
```

**Important implementation notes:**
- The `onChange` closures in DirectoryWatcher dispatch to `@MainActor` via `Task { @MainActor in }` because DirectoryWatcher fires on a utility queue but all ViewModel state is `@MainActor`.
- `scanCurrentDirectory()` creates new `ImageCaptionPair` instances with fresh UUIDs each time. So selection repair must match by `imageURL`, not by `id`.
- The `isRescanning` flag prevents the infinite loop described in RESEARCH.md Pitfall 2 (reading directory contents may trigger a .write event on some FS implementations).
- Use `[watchdog]` print prefix for all debug logging (per established `[cache]` and `[saveSelected]` conventions).
