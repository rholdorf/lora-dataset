# Phase 4: Polish - Context

**Gathered:** 2025-12-21
**Status:** Ready for planning

<vision>
## How This Should Work

The app should feel like it belongs in the macOS ecosystem — indistinguishable from Apple's own apps like Finder and Notes. When you open it, nothing should feel "off" or like it was made with a cross-platform toolkit.

The sidebar folder tree should look and behave exactly like Finder's sidebar: same disclosure triangles, same selection highlighting, same hover states. The overall app should have visual consistency throughout — no jarring mismatches in spacing, colors, or typography.

</vision>

<essential>
## What Must Be Nailed

- **Sidebar feels right** — The folder tree must look and behave exactly like Finder's sidebar (disclosure triangles, selection highlighting, hover states)
- **Overall consistency** — Everything should feel cohesive with no jarring mismatches in style across the app

</essential>

<boundaries>
## What's Out of Scope

No specific exclusions — whatever it takes to make the app feel native. Focus is on visual refinements to match macOS native apps, not new features.

</boundaries>

<specifics>
## Specific Ideas

- Finder's sidebar is the reference for the folder tree: disclosure triangles, selection highlighting, hover states
- Match the visual language of Apple's built-in apps (Finder, Notes)

</specifics>

<notes>
## Additional Context

This is the final phase of v1.1. The goal is purely visual/interaction polish — the functionality is complete, now it needs to look and feel like a first-party macOS app.

</notes>
