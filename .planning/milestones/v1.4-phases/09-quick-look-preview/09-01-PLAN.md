**3. Refactor `DatasetViewModel.swift`:**

Remove `let qlPreviewHelper = QLPreviewHelper()` and the old `quickLook(url:)` method.

Add a new `toggleQuickLook()` method:
```swift
func toggleQuickLook() {
    guard selectedPair != nil else { return }
    NSApp.keyWindow?.makeFirstResponder(nil)
    let panel = QLPreviewPanel.shared()!
    if panel.isVisible {
        panel.orderOut(nil)
    } else {
        panel.updateController()
        panel.makeKeyAndOrderFront(nil)
    }
}
```

Update `selectedID` didSet to include panel-following logic:
```swift
didSet {
    if let pair = selectedPair {
        UserDefaults.standard.set(pair.imageURL.path, forKey: "lastSelectedImagePath")
    }
    // Update QL panel if visible to follow selection
    if QLPreviewPanel.sharedPreviewPanelExists(),
       let panel = QLPreviewPanel.shared(),
       panel.isVisible {
        if selectedPair != nil {
            panel.reloadData()
        } else {
            panel.orderOut(nil)
        }
    }
}
```

Add panel close in any folder navigation method (`navigateToFolder` or wherever `pairs` is reset and `selectedID` becomes nil). The `selectedID` didSet above already handles this via the `else { panel.orderOut(nil) }` branch, but also add an explicit close in `loadDirectory()` before resetting pairs, to handle edge cases:
```swift
// At the start of loadDirectory(), before clearing pairs:
if QLPreviewPanel.sharedPreviewPanelExists(),
   let panel = QLPreviewPanel.shared(),
   panel.isVisible {
    panel.orderOut(nil)
}
```
