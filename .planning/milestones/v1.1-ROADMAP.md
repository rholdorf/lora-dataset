# Milestone v1.1: Finder Navigation

**Status:** SHIPPED 2025-12-21
**Phases:** 1-4
**Total Plans:** 5

## Overview

Transform the app from a single-directory viewer into a Finder-like dataset browser. Add a sidebar folder tree for navigation, implement save shortcuts and dirty indicators, handle security-scoped bookmarks for directory traversal, and polish to native macOS standards.

## Phases

### Phase 1: Directory Tree Navigation

**Goal**: Replace current flat file list with expandable folder tree in sidebar
**Depends on**: Nothing (first phase)
**Plans**: 2 plans

Plans:
- [x] 01-01: FileNode model and ViewModel folder tree state
- [x] 01-02: Folder tree view and navigation UI

**Details:**
- Created FileNode struct with recursive children structure
- Extended DatasetViewModel with folderTree and selectedFolderID
- Implemented OutlineGroup-based folder tree with navigation
- Resolved SwiftUI view update conflicts with Task-based state synchronization
- Used persistent security-scoped access for entire session

## Milestone Summary

**Key Decisions:**

| Phase | Decision | Rationale |
|-------|----------|-----------|
| 01-02 | No lazy loading | Build complete tree upfront to avoid SwiftUI view update conflicts |
| 01-02 | Persistent security-scoped access | Keep access active for session, required for image loading |
| 01-02 | Local @State for List selection | Avoid binding @Published directly to List selection, sync via Task |
| 01-02 | OutlineGroup for tree | Native SwiftUI tree support, simpler than DisclosureGroup recursion |
| 02-01 | Separate SaveButtonView with @ObservedObject | FocusedValue alone doesn't observe ViewModel changes |
| 02-01 | Include text fields in Equatable | SwiftUI needs to detect captionText changes for dirty indicator updates |
| 03-01 | Path strings for UserDefaults | url.path strings for persistence compatibility |
| 03-01 | Recursive DisclosureGroup | OutlineGroup doesn't expose expansion bindings |
| 04-01 | Manual disclosure over DisclosureGroup | DisclosureGroup intercepts label clicks, preventing navigation |
| 04-01 | onTapGesture over Button in List | Button hit-testing unreliable after List re-renders |

**Issues Resolved:**

- SwiftUI view update conflicts ("Publishing changes from within view updates")
- Folder first-click navigation not working (onTapGesture vs List selection)
- Dirty indicator not appearing (Equatable not including text fields)
- Cmd+S always disabled (FocusedValue observation issue)
- Folder navigation stopping after first click (Button hit-testing in List)
- DisclosureGroup intercepting folder label clicks

**Issues Deferred:**

None
