<objective>
Implement session restoration to remember and restore the last viewed image position on app relaunch.

Purpose: Users should return to exactly where they left off - same folder, same image selected - without needing to re-navigate.
Output: App remembers current folder path and selected image path, restores them on next launch.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Persist and restore current folder path</name>
  <files>lora-dataset/lora-dataset/DatasetViewModel.swift</files>
  <action>
Add persistence for `directoryURL` (the navigated-to folder, not just root):

1. In `navigateToFolder()`, after setting `directoryURL`, persist the path:
   ```swift
   UserDefaults.standard.set(url.path, forKey: "lastViewedFolderPath")
   ```

2. In `restorePreviousDirectoryIfAvailable()`, after restoring root and scanning:
   - Check if `lastViewedFolderPath` exists in UserDefaults
   - If yes, verify the path starts with the root path (security check)
   - If valid, call `navigateToFolder()` to restore the last viewed folder

3. In `chooseDirectory()`, clear `lastViewedFolderPath` when user picks a new root (fresh start).

Use the same pattern as `expandedPaths` - simple string paths in UserDefaults.
  </action>
  <verify>
1. Navigate to a subfolder, quit app
2. Relaunch - should open in the subfolder, not root
3. Select new root folder - should start fresh at root
  </verify>
  <done>
- Navigating to folder persists path to UserDefaults
- App restore reads persisted folder path
- Subfolder selection survives app restart
- New root selection clears persisted folder path
  </done>
</task>

<task type="auto">
  <name>Task 2: Persist and restore selected image</name>
  <files>lora-dataset/lora-dataset/DatasetViewModel.swift</files>
  <action>
Add persistence for selected image position:

1. Add a property to track last selected image path:
   ```swift
   private var lastSelectedImagePath: String? = nil
   ```

2. In `scanCurrentDirectory()`, after setting `selectedID = pairs.first?.id`:
   - Check if `lastSelectedImagePath` is set and matches a pair in the current folder
   - If match found, set `selectedID` to that pair's ID instead of first
   - Clear `lastSelectedImagePath` after use (one-time restore)

3. When `selectedID` changes (need to observe this), persist the image path:
   - Add a `didSet` observer or a method called when selection changes
   - Persist: `UserDefaults.standard.set(selectedPair?.imageURL.path, forKey: "lastSelectedImagePath")`

4. In `restorePreviousDirectoryIfAvailable()`, before scanning:
   - Load `lastSelectedImagePath` from UserDefaults into the property

5. In `chooseDirectory()`, clear `lastSelectedImagePath` (fresh start on new root).

Key insight: We persist the path string and match by path after scanning, since UUIDs change.
  </action>
  <verify>
1. Select an image (not the first one), quit app
2. Relaunch - should select that same image
3. Navigate to subfolder, select image there, quit
4. Relaunch - should be in subfolder with that image selected
5. Pick new root - should start fresh with first image
  </verify>
  <done>
- Selected image path persisted to UserDefaults on selection change
- App restore matches persisted path to scanned pairs
- Correct image selected after restart (not just first image)
- New root selection clears persisted selection
  </done>
</task>
